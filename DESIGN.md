# Design Mode

Use this file when user says:

- `design activate`
- `design ultra`

Load [design-tokens.md](/Users/danispreldzic/Desktop/Danis/PROJECTS/reciept/design-tokens.md) first for exact project typography, spacing, color, radius, and component defaults.

Goal:

- make UI feel intentional
- improve hierarchy before decoration
- avoid generic layouts
- keep UX clear on mobile and desktop

## Core Rules

- start from screen goal and primary action
- prefer one strong visual idea over many weak ones
- reduce clutter before adding polish
- preserve existing design system unless user wants redesign
- do not trade usability for style

## Hierarchy

- make primary action obvious
- group related content tightly
- separate unrelated content clearly
- keep one focal area per section
- use size, weight, contrast, and spacing with intent

## Spacing

- use consistent spacing rhythm
- give dense areas structure, not random gaps
- let whitespace create groups and emphasis
- avoid cramped cards and overpadded containers

## Typography

- define clear roles for heading, body, support, and label text
- use fewer font sizes with stronger hierarchy
- avoid weak contrast in secondary text
- prefer readable line lengths
- use weight and size before using many colors

## Color

- assign colors by role: background, surface, text, muted, accent, feedback
- keep accent color meaningful
- avoid noisy palettes
- use contrast to guide attention
- danger, warning, and success should read instantly

## Components

- buttons need clear priority
- input states must be obvious
- cards should not all compete equally
- empty states should guide next action
- tables and lists should scan fast

## Critique Order

When reviewing design, prioritize:

1. hierarchy
2. layout and spacing
3. typography
4. color usage
5. interaction states
6. polish

## Design Ultra

When user says `design ultra`:

- be more opinionated
- reject bland or template-looking solutions
- push stronger typography, contrast, and composition
- still preserve usability and existing product constraints
